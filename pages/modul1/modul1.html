Wprowadzenie do CSS

Kaskadowe arkusze stylów, w skrócie CSS jest to język służący opisowi formy prezentacji stron www.

CSS opisuje jak elementy html, powinny być prezentowane na ekranie, podczas wydruku czy w czytniku ekranowym...

Obecnie mamy trzy standardy CSS:

  • CSS w wersji CSS1 , obecnie mocno przestarzały
  • CSS w wersji CSS2.1 , obecnie rekomendowany zgodnie z "developer.mozilla.org"
  • CSS w wersji CSS3 , obecnie rozbity na mniejsze moduły w ciągłym procesie standaryzacji.

Idąc dalej za pomocą CSS, możemy na przykład zmienić font na stronie, dodać animację do elementu HTML czy manipulować odstępami na stronie.

W celu dodania pliku ze stylami do strony, powinnismy umiescić <link rel="stylesheet" href="mojPlik.css"> między znacznikami <head>...<head>


<!DOCTYPE html>
<html>
  <head>
    <meta charset="utf-8">
    <title>My CSS experiment</title>
    <link rel="stylesheet" href="style.css">
  </head>
  <body>
    <h1>Witam serdecznie!</h1>
    <p>To mi wygląda jak pierwszy przykład :)</p>
  </body>
</html>

Style w "style.css"


h1 {
    color: blue;
    background-color: yellow;
    border: 1px solid black;
}

p {
    color: red;
}

Witam serdecznie!

To mi wygląda jak pierwszy przykład :)

Kolejnym rozwiązaniem dla szybkiego prototypowania może być dodanie styli w znacznikach <style>...<style>, również w elemencie "head" 


<style>
    [my-2] {
        background: red;
    }
</style>

Poza dodawaniem styli do pliku HTML w osobnym pliku, możemy również dodawać pliki bezposrednio do elementów html w pliku HTML (Inline style). Jednak tego typu zabiegi, nie są mile widziane i powinny być stosowane z rozwagą lub o ile to możliwe nie powinny być implementowane wcale w ten sposób... 


<!DOCTYPE html>
<html>
    <head>
    <title>Playing with Inline Styles</title>
    </head>
    <body>
    <p style="color:blue;font-size:46px;">
        I'm a big, blue, <strong>strong</strong> paragraph
    </p>
    </body>
</html>

Trzy zasady pisania dobrego kodu

Przed klepaniem kodu, zawsze dobrze jest zastanowić się co tak na prawdę chcemy pisać i jak zrobić to najlepiej.

Tworzymy jedną stronę, która swietnie wyswietla się na wszystkich urządzeniach. Mam tu namysli małe wyswietlacze np. telefonów przez tablety na dużych monitorach kończąc.
Nasz kod powinien być czysty (wliczając w to odpowiednie formatowanie), łatwy do zrozumienia oraz możliwy do zastosowania w innym projekcie bez kopiowania tony zbędnego badziewia...
Wydajnosć jest jednym z kluczowych elementów wpływających na odbiór strony przez potencjalnego Klienta. Nasze strony powinny być ładne, funkcjonalne i przedewszystkim szybkie/wydajne

Selektory CSS

W tym paragrafie skupimy się na najważniejszych selektorach. Nie będziemy jednak przedstawiać tu wszystkich możliwych selektorów, bo jest ich zwyczajnie zbyt dużo.

Zaczynając od podstaw selektory możemy podzielić na kilka grup:

  • element: selektorem w tym przypadku może być np: p, button, a itp.
  • klasa: selektorem w tym przypadku może być np: .mojaSuperKlasa, .dark itp.
  • id: selektorem w tym przypadku może być np: #mojElementHtml, #light itp.
  • atrybut: selektorem w tym przypadku może być np: [data-parent$='fundamentals'], input[type=text], [class|=top], [title~=xyz], [class*="Dlu"] itp. 


<div id="collapseOne" class="collapse show" data-parent="#fundamentals"></div>
<input type="text" class="form-control" name="userName" required>
<p class="top-border"></p>
<img src="./xx/yy/zz.jpg" alt="some image" title="innaKlasa xyz yyy">
<div class="bardzo Dluga Klasa">Some text</div>

Dodatkowo poza podstawowymi selektorami, możemy zastosować selektory łączone w poniższych kombinacjach

  • Dla przykładu div + p, zaznaczy pierwszy element p zaraz po div
  • Dla przykładu p.zero ~ p, pominie paragraf z klasą p.zero i zaznaczy wszystkie paragrafy poniżej
  • Dla przykładu div > p, zaznaczy wszystkich bezposrednich potomków kontenera div
  • Dla przykładu * zaznaczy wszystko co dostępne na stronie ;)

<p class="zero">paragraf numer 0</p>

<div class="kontener">
    Content in div here
    <p class="srodek">paragraf w div</p>
</div>

<p class="jeden">paragraf numer 1</p>    
<p class="dwa">paragraf numer 2</p>

Żeby tego było mało do dyspozycji mamy jeszcze selektory pseudoklas

  • Dla przykładu :nth-child(2), zaznaczy drugie dziecko swojego rodzica .kontener > p:nth-child(2) {}
  • Dla przykładu p:last-of-type, zaznaczy ostatni paragraf w .kontener jak i w globalnym kontenerze
  • Dla przykładu :checked, pozwala ostylować zaznaczony element checkboxa
  • Dla przykładu :hover, pozwala ostylować odnosnik podczas mouseover

<p class="zero">paragraf numer 0</p>

<div class="kontener">
    Content in div here
    <p class="srodek">paragraf w div</p>
    <p class="srodek">paragraf2 w div</p>
</div>

<p class="jeden">paragraf numer 1</p>    
<p class="dwa">paragraf numer 2</p>

Elementy blokowe, liniowe

Ze względu na ilość zajmowanego miejsca elementy w języku html możemy podzielić na liniowe oraz blokowe. Elementy liniowe zajmą tylko tyle miejsca ile same potrzebują, pozostawiając miejsce na tej samej linii dla kolejnych elementów liniowych. Natomiast elementy blokowe zawsze wyświetlane są na nowej linii i zajmują jej całą powierzchnie, a kolejne elementy ukazywane są pod spodem, na kolejnej linii.

  • Elementami liniowymi są <a>, <span>, <em>, <i>, <img>, <strong>

Przykładami elementów blokowych, czyli takich, które zajmują całą linię, mogą być.

  • Elementy blokowe <h1 - h6>, <div>, <p>, <ul>

Pozycjonowanie i rozkład elementów na stronie

Pozycjonowanie w CSS pozwala nam dokładnie okreslić miejsce elementu na stronie. Elementy mogą być rozmieszczane w relacji względem swojej pozycji, absolutnie względem elementu nadrzędnego jak i również posiadać pozycję ustaloną względem okna przeglądarki.

Metody pozycjonownia - zastosowanie własciwosci "position"
Za pomocą property position możliwe jest ustalenie pozycji w relacji do elementu poprzedzającego, pozycji absolutnej względem elementu przodka lub pozycji ustalonej względem okna przeglądarki. W celu zastosowania tych możliwosci możemy zastosować.
  • static : Jest to podstawowa opcja przypisana na start dla każdego elementu. Zastosowanie, którejs z opcji left, right, top, bottom, z-index jest... bezcelowe :)
  • relative : Jest podobny do property static, element pozostaje nadal w normalnym położeniu ale zastosowanie left, right, top, bottom, z-index jest... zasadne :)
  • absolute : To co tygryski lubią najbardziej ;), dzięki zastosowaniu tej własciwosci element jest "wyjęty" z normalnego rozkładu w dokumencie. Dzięki temu element wygląda jak by był postawiony warstwę wyżej left, right, top, bottom, z-index jest... zasadne :)
  • fixed : Jest podobny do property absolute, element jest również "wyjęty" z normalnego rozkładu w dokumencie. Różnica polega na tym, że jest on pozycjonowany bezposrednio względem okna przeglądarki i pozostaje w tym samym miejscu podczas scrollowania strony.
  • sticky (props experymentalny, należy sprawdzać przeglądarkę, z którą będziemy go używać) : element jest traktowany jak normalny relatywny element, do momentu kiedy nie osiągnie górnej bariery przeglądarki, lub elementu scrollowanego, w którym jest umieszczony.

Jednostki miary w CSS

Stare dobre "px" są nadal dobre ;), ale dobre można zamienić na lepsze. Szczególnie powinnismy rozważyć zmianę starych jednostek miary na "rem" przy projektowaniu serwisów responsywnych. Ułatwi to skalowanie odległosci, wielkosci itp, za pomocą zmiany jednego parametru...

Poniżej zapoznamy się z istotnymi jednostkami, które pomogą rozwiać tajemnicze symbole jak "rem", "am" itp.

Jednostka "em" jest to jedna z jednostek względnych. Jej rozmiar zależy od rozmiaru czcionki zastosowanej bezposrednio lub odziedziczonej od rodzica, który definiował zmianę wielkosci fontu. Dla elementu, którego czcionka wynosi 20px, 1 em oznacza 20px. Jeśli użytkownik korzysta z dużej czcionki (np. na dużym monitorze) albo małej czcionki (np. na urządzeniu mobilnym), to podane wielkości będą proporcjonalne.

Przykład zastosowania


.mojDiv {
    margin: 1em; // odpowiada wartosci 20px
    line-height: 1.2em; // odpowiada 24px, bo 20 * 1.2 = 24...
}

Dodanie jednostek "em" nie było ostatecznym słowem i w 2013 do css dodano "root em", co finalnie przełożyło się na nazwę "rem". Jednostki "rem", są bardzo podobne do "em". Podstawowa różnica pomiędzy nimi polega na tym z jakiego miejsca brana jest referencja wielkości czcionki. Dla "em" jest to element, bieżący lub najbliższy wrapper, który definiował wielkość czcionki natomiast dla "rem", jest to zawsze wielkość zdefiniowana jako wielkość tzw. bazowej czcionki dokumentu

Inne nowe jednostki pozwalają zapisać reguły zależne od rozmiaru okna użytkownika. Mowa o "vw" i "vh". Ten pierwszy stanowi jedną setną szerokości okna, natomiast drugi jedną setną wysokości. Dodano także jednostkę vmin, odpowiadającą mniejszej z wartości vw i vh oraz vmax

    Poniżej znajdzie się lista przykładów, dotycząca zasad rozwiązywania konfliktów, jako pierwszy przedstawię przykład z propsem !important

    color: #fff;
}

/*Wynik: (0, 1, 2, 1)*/

    color: #dedede;
}

/*Wynik: (0, 0, 0, 1)*/
a {
    color: #00ffee
}

    Jeżeli zdaży się, że mamy selektory elementów z taką samą iloscią punktów, przypisany zostanie ten styl, który był renderowany pózniej

Animacje CSS

W tym rozdziale, chcę przybliżyć możliwosć stosowania animacji w czystym CSS. Koncepcja jest zbliżona do animacji np, w Flashu :). Definiujemy swoją linię czasu, gdzie pózniej dodajemy swoiste keyframes z podziałką procentową...

Pierwszy krok polega na zdefiniowaniu "timeline" oraz nadanie odpowiedniej nazwy :). W pliku css, definiujemy przykładową animację używając słowa kluczowego "@keyframes" 


@keyframes mojaAnimacja {
    0% {
        opacity: 0;
    }

    50% {
        opacity: .8;
    }

    100% {
        opacity: 0;
    }
}

W celu zastosowania przygotowanej animacji, do stylów animowanego elementu dodajemy następujące własciwosci


.animowanyElement {
    animation-name: mojaAnimacja
    animation-duration: 3s;
}

Przykład troszkę bardziej skomplikowanej animacji


@keyframes mojaAnimacja {
    0% {
        opacity: 0;
    }

    50% {
        opacity: .8;
    }

    100% {
        opacity: 0;
    }
}
                                

@keyframes mojaAnimacja2 {
    0% {
        opacity: 0;
        left: 0;
    }

    25% {
        opacity: 1;
        left: 90%;
    }

    50% {
        left: 0;
    }

    75% {
        opacity: 0.5
    }

    100% {

    50% {
        transform: rotate(0deg);
    }

    90% {
        transform: rotate(360deg);
    }

Animacja ma kilka własciwosci, które pozwalają kontrolować przebiegiem animacji

  • animation-delay: własciwosc ta pomaga opóznic start animacji
  • animation-iteration-count: własciwoscią tą, możemy zapętlić animację n razy, dla ciągłej animacji możemy zastosować wartosć "infinite"
  • animation-direction: Za pomocą tej własciwosci, możemy odtworzyć animację od tyłu.

RWD

Responsive web design (RWD) – jest to technika projektowania strony www, tak aby jej wygląd i układ dostosowywał się automatycznie do rozmiaru okna przeglądarki, na której jest wyświetlany np. przeglądarki, smartfonów czy tabletów. Strona tworzona w takiej technice jest uniwersalna i wyświetla się dobrze zarówno na dużych ekranach, jak i na tych mniejszych.

Obsługa tej techniki odbywa się za pomocą media queries, które pozwalają rozpoznać rozdzielczość okna przeglądarki i zastosować odpowiednie style css.

Domyslam się, że nie jednokrotnie widzieliscie już poniższy kawałek kodu


<meta name="viewport" content="width=device-width, initial-scale=1" />

Właściwość width (szerokość) kontroluje wielkość widocznego pola. Można jej przyisać konkretną liczbę np. width=600, albo specjalne słowo kluczowe - device-width, które oznacza nadanie wielkości ekranu w pikselach przy zoomie na poziomie 100%. (Istnieją również height i device-height, które przydadzą się w przypadku gdy wielkość danego elementu zależy od wysokości ekranu.) Właściwość initial-scale kontroluje poziom zbliżenia po załadowaniu strony.

Poniżej przykład zastosowania media queries


@media screen and (max-width: 400px) {
    a.button {
        background-color: red;
        color: #fff;
        font-size: 2rem;
    }
}

Architektura CSS, zaznajomienie się z BEM

BEM - "Block Element Modifier" jest to jedna z konwencji nazewnictwa klas css stosowanych do poszczególnych elementów składowych strony.

Zasady ogólne BEM skłaniają nas do używania tylko klass, nie powinno stosować się identyfikatorów czy nazw elementów

W BEM stosujemy następującą konwencję do budowy nazw klass przypisanych do elm, html

  • .block: Pierwszy człon opisuje blok ( może to być logiczna nazwa danej sekcji kodu )
  • __element: Drugi człon poprzedzony dwoma "podkreślnikami" oznacza, że dana klasa dotyczy danego elementu
  • --modifier: Trzeci człon poprzedzony dwoma myslnikami oznacza, klasę modyfikatora

Przedstawioną konwencję zastosujemy do poniższego kawałka kodu


<section>
    <input type="text" />
    <input type="password"/>
    <button type="button">Cancel</button>
    <button type="submit">Submit</button>
</section>

Po zastosowaniu BEM


<section class="loginComponent">
    <input class="loginComponent__input" type="text" />
    <input class="loginComponent__input loginComponent__input--passwd" type="password"/>
    <button class="loginComponent_button" type="button">Cancel</button>
    <button class="loginComponent_button loginComponent_button--submit" type="submit">Submit</button>
</section>

Oraz zastosowane style


.loginComponent {
    width: 300px;
    height: 200px;
    border: 1px solid #dedede;
}

.loginComponent__input {
    background-color: #dedede;
    border-radius: 5px;
    border: 1px solid #666666;
}

.loginComponent__input--passwd {
    border: 1px solid #red;
}

.loginComponent_button {
    width: 60px;
    text-align: center;
    background-color: #dedede;
}

.loginComponent_button--submit {
    background-color: #4CAF50;
