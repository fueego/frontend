Wprowadzenie do SASS

Sass, less i inne przyprawy... Co to jest sass, najprosciej to taki trochę CSS na sterydach :), zwany dalej preprocesor języka CSS. Straszne słowo preprocesor... Generalnie kod napisany w Sassie musi być przekompilowany do zwykłego CSS. Przeważnie przy większych projektach konwertowaniem plików *.scss do *.css zajmuje się webpack ( czasem gulp ), które przy okazji robią wiele innych przydatnych rzeczy. Jednakże, "setup" takiego srodowiska, wykracza poza ramy tego kursu.

My do tego celu zastosujemy programik o nazwie "koala", o którym mowa będzie w następnym dziale.

Wracając do samego Sass-a, pisanie arkuszy stylów w Sassie wznosi nas na wyższy poziom. Jest to dalej CSS ale możemy w łatwy sposób posiłkować się zmiennymi, korzystać z zagnieżdżeń styli, tworzyć funkcje, które możemy implementować do inncyh klass. Co więcej Sass, pozwala nawet na importowanie styli z zewnętrznego pliku ( podobnie jak moduły w JS ).

Bezbolesna kompilacja scss do css

Jak wczeniej wspomniałem do kompilacji plików sass-a będziemy stosowali program "Koala". Jest to program, który może być zainstalowany na Windowsie, Linuxie oraz Macu.

Kompilacja Sassa za posrednictwem Koali, jest dziecinnie prosta. Wystarczy dodać folder, z zródłem *.scss. Zaznaczając plik z listy po prawej stronie mamy listę, możliwych opcji jakie chcemy zastosować podczas kompilacji.

W celu przypisania scieżki, w której będziemy przechowywać pliki css, należy kliknąć na pliku z listy prawym klawiszem myszki, z dropdown-a wybrać opcję "Set Output Path"

Pierwsze kroki, zmienne i zagnieżdżanie

W tradycyjnym CSS, nie mamy możliwosci stosowania zmiennych, co szczególnie odczuwalne jest gdy chcemy zmienić kolorystykę elementów na stronie, lub charakterystyczny odstęp od sekcji w głównym kontenerze strony itp.

Poniżej przykład zwykłego CSS


.mainContent a {
    color: #fefefe;
    font-size: 18px;
    margin-top: 15px;

    background-color: #fffeee;
    border-radius: 5px;
    border: 2px solid #dedede;
}

.mainContent .button {
    border-radius: 10px;
    background-color: #dedede;
    borderd: 3px solid #000;
}

.footer li > a {
    color: #fff;
    background: #000;
    padding: 10px;
    border: 1px solid #fff;
    border-radius: 3px;
}

Zakładamy czysto teoretycznie, że chcemy ujednolicić kolor wszystkich odnosników wraz z klasami .button, dodatkowo chcemy mieć tą samą wartosć border-radius dla wszystkich elementów a. Za pomocą Sassa, osiągniemy to w łatwy i przejrzysty sposób. 


/*Zmienne najlepiej trzymać w oddzielnym pliku, pogrupowane w logiczne skecje*/
/*Jeżeli nie mamy wielu arkuszy stylów, możemy trzymać to w jednym pliku*/

/*Colors*/
$link-background: pink;

/*Size*/
$border-radius: 2px;

.mainContent a {
    color: #fefefe;
    font-size: 18px;
    margin-top: 15px;

    background-color: $link-background;
    border-radius: $border-radius;
    border: 2px solid #dedede;
}

.mainContent .button {
    border-radius: $border-radius*2;
    background-color: $link-background;
    borderd: 3px solid #000;
}

.footer li > a {
    color: #fff;
    background: $link-background;
    padding: 10px;
    border: 1px solid #fff;
    border-radius: $border-radius;
}

Sass to nie tylko zmienne, bardzo przydatną funkcjonalnoscią jest zagnieżdżanie


.wrapper .mainMenu .headSection a {
    color: #fff;
    background: #000;
}

.wrapper .mainMenu .headSection .subHeader span {
    color: #000fff;
    padding: 10px;
}

.wrapper .mainMenu .footerSection a {
    margin-bottom: 5px;
    color: red;
}

.wrapper .mainMenu .footerSection a:hover {
    margin-bottom: 5px;
    color: red;
}

Dobrze znamy kilometrowe grupy selektorów. Generalnie ten styl ułatwia identyfikacjię. Po pierwszym rzucie oka, widzimy że to rodzina styli dotyczących bloku .mainMenu, ale można to zrobić łatwiej i czytelniej 


.wrapper {
    .mainMenu {
        .headSection {
            a {
                color: #fff;
                background: #000;
            }

            .subHeader {
                span {
                    color: #000fff;
                    padding: 10px;
                }
            }
        }

        .footerSection {
            a {
                margin-bottom: 5px;
                color: red;

                &:hover {
                    margin-bottom: 5px;
                        color: red;
                }
            }
        }
    }
}

Mixiny, Extendy i Funkcje

  • Mixiny: Mixiny to nic innego jak małe kontenerki styli, które możemy z powodzeniem dodawać do innych klas. W mysl reguły "DRY"
  • Funkcje: to nic innego jak wbudowane dodatkowe funkcjonalnosci, jak np lighten, darken, grayscale oraz wiele więcej. Jeżeli mamy taką potrzebę, możemy sami napisać własną funkcję
  • Extendy: zbliżone są w swoim działaniu do mixinów, jednak nie kopiują one kodu ( w wygenerowanym kodzie css ), a selektor. W wyniku czego mamy mniejszy plik wynikowy.

Wyobrazmy sobie prostą sytuację, chcemy dodać .border-radius do linków, w nawigacji głównej oraz w footerze. Nie będziemy się duplikować więc zastosujemy do tego Mixiny :) 


.mainNav .nav a {
    border: 1px solid #000;
    padding: 10px;
    margin: 10px;
    display: block;
    border-radius: 5px;
}

...

.footer .footerNav a {
    border: 1px solid #000;
    padding: 5px;
    border-radius: 3px;
}

Smentnie to wyglądało, ale poniżej zrobimy mały refaktoring ;)


@mixin borderRadius($size) {
    -webkit-border-radius: $size;
    -moz-border-radius: $size;
    border-radius: $size;
}

.mainNav .nav a {
    border: 1px solid #000;
    padding: 10px;
    margin: 10px;
    display: block;
    @include borderRadius(5px);
}

...

.footer .footerNav a {
    border: 1px solid #000;
    padding: 5px;
    @include borderRadius(3px);
}

Funkcje w Sasie, to dodatkowe możliwosci jakie daje nam ten preprocesor, jeżeli mam być szczery, nigdy ich nie stosowałem, szczerze to wczesniej myslałem, że mixiny to to samo jak i funkcje.

Funkcje w zastosowaniu są jednakowe jak i mixiny, bez @include. np: background: lighten($color, 10%) 


$black-bg: #000;

.mainModule a {
    background: lighten($black-bg, 25%); /* #404040 */
}

Jak widać na powyższym przykładzie kolor czarny został zamieniony na #404040. Poniżej przedstawię przykład własnej funkcji 


@function add($l1, $l2) {
    @return $l1 + $l2;
}

.mainModule a {
    border: add(10px, 5px) solid #dedede;
}

Extendy, współpracują scisle z "placeholderami". Przykład html.


<div class="mainModule">
    <a>
        Some Span here
    </a>
</div>
    
<div class="footer">
    <a>
        Some additiona text
    </a>
</div>

Style *.scss


%button-stailo {
    border: 1px solid #dedede;
    background-color: #000;
    color: #fff;
}

.mainModule a {
    @extend %button-stailo;
}

.footer a {
    @extend %button-stailo;
}

Wynikowy *.css


.mainModule a, .footer a {
    border: 1px solid #dedede;
    background-color: #000;
    color: #fff;
}

Architektura 7-1 CSS przy użyciu Sass

Zastosowanie wzorca 7 do 1, przy zastosowaniu Sassa, zanjduje się najlepiej w implementacji przy dużych projektach. Przed tym jednak musimy dowiedzieć się jak importować zewnętrzne pliki *.scss

Jak to wszystko w Sass, import pliku odbywa sie w dosć prosty sposób, wystarczy tylko w głównym pliku *.scss zaimportować zewnętrzny plik *.scss, który swoją nazwę zaczyna od "podkreslnika" np: "_color.scss" "_mixin.scss" 


@import "color";
@import "mixin";
...

Teraz gdy importowanie nie stanowi tajemnicy, przejdzmy do meritum.

Cała logika tego postępowania polega na podzieleniu kodu *.scss na 7 filarów, które to zas stanowią merytoryczny podział arkuszów styli odwołujących się do poszczególnych częsci strony

Wszystkie "częsci" kodu *.scss są następnie ładowane w głównym pliku main.scss
